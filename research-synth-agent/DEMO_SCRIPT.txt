[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a quick look under the hood. 
At the center, we have our Research Synth Agent—a powerful LlmAgent built on Gemini 3.5 Flash that drives the entire workflow. 
Instead of relying on outdated internal knowledge, it reaches out to the real world using a local MCP Server. 
The MCP Server gives the agent three specific superpowers, or tools: it can search arXiv for open-access pre-prints, it can query the Semantic Scholar Graph API for broader metadata, and if those fail or the topic is too recent, it acts as a safety net by querying DuckDuckGo for live web snippets.
Once the agent gathers the raw JSON data from these endpoints, it synthesizes the information and returns it to the user.

If I asked it a broader question—like "What is the latest news regarding Google AI announcements this week?"—it would realize the topic isn't strictly academic, gracefully catch that exception, and automatically pivot to the DuckDuckGo tool to fetch the latest news articles instead. 

[3:00 — SECURITY & MCP]
One of the core strengths of this architecture is security and stability. The MCP server isolates the tool execution from the agent's brain. If an API key is missing or an endpoint goes down, the tools handle the exceptions natively and fallback gracefully without crashing the whole application. The agent simply receives an error context and tries a different path.
